## What this project is

A tiny localStorage-backed ToDo app (Vite + React 19 + TS). The app code is the _substrate_, not the goal â€” most of the value lives in `.claude/`, `.github/`, and `.mcp.json`.

## Stack

- **Frontend:** React 19, TypeScript 6, Vite 8
- **Tests:** Vitest 4 + Testing Library (unit), Playwright (e2e)
- **Quality:** ESLint flat config, Prettier
- **Package manager:** npm (lockfile committed)
- **Node:** 20

## Conventions

- Functional components only; named exports preferred (App still uses default export for `main.tsx` import compatibility)
- No `any` â€” ESLint enforces this
- Tests live next to source as `*.test.ts(x)`
- Module organisation: `components/` (UI), `hooks/` (state/logic), `lib/` (pure utilities)
- Use `crypto.randomUUID()` for IDs

## Where things live

## Don't touch (without asking)

- `.claude/hooks/*.mjs` â€” these run automatically; review carefully before changing
- `.github/workflows/claude.yml` â€” generated by `/install-github-app`; manual edits get clobbered
- `package-lock.json` â€” regenerate via npm, don't hand-edit

## Tips for working here

- Run `/review-local` before pushing
- Run `/coverage-gaps` to find untested files
- After editing `*.ts` or `*.tsx`, the post-edit hook runs Prettier automatically
- The pre-bash hook blocks `rm -rf` patterns â€” for safety experiments, edit the regex in `.claude/hooks/block-rm-rf.mjs` first
